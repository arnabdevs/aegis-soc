## What's new in v4 (vs v3)

| Upgrade | v3 | v4 |
|---|---|---|
| Password hashing | SHA-256 (fast, crackable) | **bcrypt** work-factor 12 (slow by design) |
| Database | In-memory (lost on restart) | **Supabase PostgreSQL** (persistent, free) |
| Cache / Rate limiter | In-memory | **Upstash Redis** (persistent across restarts) |
| Email verification | None | **Verify-email link on register** |
| DDoS protection | None | **Cloudflare proxy** (real IP forwarding) |

## Deploy in 4 steps

### Step 1 — Free accounts to create first

| Service | URL | What for |
|---|---|---|
| Supabase | supabase.com | Free PostgreSQL database |
| Upstash | upstash.com | Free Redis (rate limiting + cache) |
| Cloudflare | cloudflare.com | Free DDoS protection |
| Render | render.com | Backend hosting |
| Vercel | vercel.com | Frontend hosting |

### Step 2 — Get your connection strings

**Supabase:**
1. Create project at supabase.com
2. Go to **Settings → Database → Connection string → URI**
3. Copy the `postgres://...` string → this is `SUPABASE_DB_URL`

**Upstash:**
1. Create database at upstash.com → Redis
2. Go to **Connect → .env**
3. Copy `REDIS_URL=redis://...` value

### Step 3 — Push to GitHub and deploy

```bash
git init
git add .
git commit -m "AEGIS SOC Engine v4.0"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/aegis-soc-v4.git
git push -u origin main
```

**Render:**
1. New Web Service → connect repo
2. Render auto-reads `render.yaml`
3. Set environment variables (see table below)
4. Deploy

**Vercel:**
1. New Project → import repo
2. Root Directory → `frontend`
3. Framework: Other → Deploy

### Step 4 — Put Cloudflare in front of Render

1. Add your Render URL to Cloudflare as a CNAME (or use a custom domain)
2. Enable **Proxy** (orange cloud) → Cloudflare now absorbs DDoS
3. All real visitor IPs are forwarded via `CF-Connecting-IP` — already handled in `app.py`

## How the 5 upgrades protect users

### 1. Cloudflare (DDoS)
All traffic hits Cloudflare's network before reaching Render.
Volumetric floods are absorbed at Cloudflare's edge — your server never even sees them.
Real visitor IPs are passed via `CF-Connecting-IP` and read in `app.py`.

### 2. Supabase PostgreSQL (data persistence)
User accounts, monitored sites, blocked IPs, scan logs — all stored in PostgreSQL.
When Render restarts (which happens on the free tier), nothing is lost.
Tables are created automatically by `init_db()` on first boot.

### 3. bcrypt (password safety)
SHA-256 hashes a password in microseconds — trivial to brute-force with a GPU.
bcrypt with work-factor 12 takes ~250ms per attempt — makes brute-forcing impractical.
If your database is ever leaked, attacker must spend months cracking each password.

### 4. Upstash Redis (rate limiting)
Without Redis, rate-limit counters live in memory and reset on every restart.
Redis keeps them persistent — a user who hits 500 req/day can't bypass it by waiting for a restart.
Also makes the scan result cache persistent across restarts.

### 5. Email verification (spam prevention)
Without verification, anyone can register with `ceo@apple.com` or any fake email.
Verification ensures only the real owner of an email address can use it.
Verification link is a 32-byte cryptographically random URL-safe token.

## Local development

```bash
cd backend
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# Add your keys to .env
# Leave SUPABASE_DB_URL and REDIS_URL blank to use in-memory mode

python app.py
# → http://localhost:5000
```

> Works perfectly without Supabase or Redis — falls back to in-memory automatically.
